# Handicap Handbook

A coffee-table-book-style Progressive Web App for a four-course golf trip on the South Carolina Grand Strand — **Blackmoor**, **True Blue**, **Arrowhead**, and **King's North at Myrtle Beach National**.

Designed to be installed to a friend's iPhone home screen, used like a native app, and read on the cart.

```
out/                ← static export — deploy this anywhere that serves static files
content/            ← pre-researched per-course JSON (do not regenerate)
public/             ← icons, manifest, downloaded imagery (generated by pipeline)
app/                ← Next.js App Router pages
components/         ← editorial primitives + map + cache warmer
lib/                ← typed content + geometry + weather loaders
scripts/            ← pipeline scripts (image fetch, geometry, icons, static maps)
```

---

## First-time setup

```sh
pnpm install
pnpm fetch-images          # downloads + resizes course imagery (~30s)
pnpm extract-geometry      # generates placeholder hole coordinates
pnpm exec tsx scripts/generate-icons.ts   # builds PWA icons from SVG
```

Then:

```sh
pnpm dev                   # http://localhost:3000
pnpm build                 # static export → ./out
```

To preview the production build locally:

```sh
pnpm build
cd out && python3 -m http.server 4242
```

## Mapbox token

The hole-detail page uses Mapbox GL JS for the satellite map and SVG overlay. Without a token, you see a polite "add a token" placeholder — the rest of the app works.

1. Sign up for a free Mapbox account at <https://www.mapbox.com/>.
2. From the [Account → Access Tokens](https://account.mapbox.com/access-tokens/) page, copy your default public token (or create a new one for this project).
3. **Lock the token to your deployment domain** before sharing the install link:
   - In the Mapbox dashboard, edit the token and add a URL restriction matching your Vercel domain (e.g. `https://handicap-handbook.vercel.app`).
   - This prevents the token from being abused if anyone inspects the page source.
4. Add it to your environment:
   - Local development: copy `.env.example` to `.env.local` and paste in your token.
   - Vercel: in the project's **Settings → Environment Variables**, set
     - `NEXT_PUBLIC_MAPBOX_TOKEN` (runtime — required)
     - `MAPBOX_TOKEN` (build-time — optional, only needed for offline static maps)

### Offline static map fallback

When `MAPBOX_TOKEN` is set, you can pre-render a satellite snapshot of every hole at build time so the map still works on the course with bad signal:

```sh
MAPBOX_TOKEN=pk.xxx pnpm static-maps
```

This downloads one ~150KB PNG per hole (81 total) into `public/images/<course>/holes/<hole>-map.png`. The `HoleMap` component falls back to these when GL JS fails to load. Free-tier Mapbox is more than enough for one full generation (50k/month limit; the build uses 81).

## Deploying to Vercel

The simplest path:

```sh
pnpm dlx vercel
```

Follow the prompts to link the project. Set `NEXT_PUBLIC_MAPBOX_TOKEN` in the Vercel environment-variables UI before promoting to production.

The `vercel.json` in this repo configures the framework, output directory, and caching headers for the service worker and image assets.

### Sharing the install link

Once deployed, the install flow on iOS is:

1. Open the Vercel URL in **Safari** (not Chrome — Chrome on iOS can't install PWAs).
2. Tap the share icon → **"Add to Home Screen"**.
3. The app launches in standalone mode (no Safari chrome) with the brass-and-cream icon.
4. On first launch with WiFi, the home page eagerly warms the cache — wait ~30 seconds for the "✓ Offline-ready" chip before going off the WiFi.
5. After that, the entire app is usable in airplane mode.

## The imagery pipeline

`scripts/fetch-images.ts` reads `content/imagery/<course>.json` (entries with `source_page_url`), resolves the actual image URL via two methods:

1. **Direct fetch** for CDN URLs that already point to a `.jpg`/`.png`/`.webp`.
2. **HTML scraping** with Cheerio for source-page URLs. Picks the best candidate using og:image, alt-text matching, hole-number hinting, WordPress size-suffix heuristics, and a logo blacklist. Falls back to **Playwright (headless Chromium)** for JS-rendered pages.

Each downloaded image is validated against a minimum dimension (400×400 landscape, or 250×380 portrait for hole-card images) and a minimum byte count to catch placeholder pixels. Two webp variants are generated per image (`@1280` and `@640`).

Output: `content/imagery/<course>.resolved.json` with local paths.

The pipeline is idempotent — already-downloaded images are skipped. Re-run after editing the source manifest.

## The hole geometry pipeline

`scripts/extract-hole-geometry.ts` generates `content/geometry/<course>.json` with approximate tee and green coordinates per hole. V1 uses course-center coordinates with a circular distribution per hole — enough for the map to feel hole-specific.

To refine: edit the generated JSON directly. Click the tee and green of a hole in Google Maps or Mapbox, copy the `[lat, lon]` pair into the corresponding entry, and flip `is_placeholder` to `false`. The `HoleMap` component will use exact coordinates the next time it renders.

## Rights & credits

All imagery is **course-provided** (official sites), **editorial** (golf travel publications), or **aggregated** (GolfPass, Golf Digest, etc). Each entry in `content/imagery/<course>.json` carries `attribution` and `rights_note` fields documenting source and reuse status.

This app is built as a **personal gift** for a four-course golf trip. Any future commercial use — including the planned hardback-book extension — must clear image rights through the respective rights holders before reuse. See `research_notes` in each course JSON for source provenance.

Editorial copy (history, hole descriptions, local knowledge, reviews) was assembled from public web sources at research time and lightly edited for tone consistency. Source URLs are listed in each course's `sources` array.

## What's still loose

- **Mapbox static map fallbacks**: deferred until a token is provided. Once set, run `pnpm static-maps` once before deploy.
- **Per-hole geometry coordinates**: all 81 holes currently use placeholder positions derived from course-center coords. Refine individual entries in `content/geometry/<course>.json` to get exact tee/green positions on the satellite map. The map already renders usefully without this — refinement is a polish task, not a blocker.
- **Two clubhouse photos** (True Blue, Arrowhead): both sources gated. UI degrades to text-only sections.
- **King's North aerial variety**: the mbn.com gallery URLs in the imagery manifest are SPA stubs without scrapable content. One large aerial from another source covers the hero; if more variety is wanted, do a fresh research pass for direct-URL imagery.

## Stack

- Next.js 14 App Router · TypeScript strict · static export (`output: 'export'`)
- Tailwind CSS 3 with custom editorial theme (cream / ink / forest / brass)
- Fraunces + Inter via `next/font/google`
- `@serwist/next` for service worker, runtime caching, offline fallback
- Mapbox GL JS for hole-detail satellite + SVG overlay
- Open-Meteo for live weather (no key required)
- Sharp + Cheerio + Playwright for the imagery pipeline
